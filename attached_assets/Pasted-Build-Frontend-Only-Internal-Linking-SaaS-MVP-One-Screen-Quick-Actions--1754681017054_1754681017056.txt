Build Frontend Only — Internal Linking SaaS (MVP, One-Screen, Quick Actions)
Только фронтенд. Без бэка, всё на mock-данных + localStorage. Никаких попапов/модалок — весь флоу проекта на одном экране.

Техбаза (на твоё усмотрение)
React + TypeScript + Vite + Tailwind (компактная типографика), Router.
Слой mockApi (in-memory + localStorage) для эмуляции данных и «прогресса».

Шапка и футер
Header (sticky): слева логотип, справа аватар с дропдауном:
– Настройки (открывает встроенную панель на странице, без модалок) → «Смена пароля» (email, старый, новый, повтор нового, кнопка «Сохранить», просто эмулируем).
– Выход (чистит localStorage «token», редирект на /login).

Footer (sticky, компакт): © ProjectName · build v0.x · ссылка «Документация» (пока #).

Маршруты (Frontend only)
/login — Вход/Регистрация (эмулируем; при успехе кладём token в localStorage).

/projects — список проектов (плоский список).

/project/:id — основной единственный экран проекта (всё, что ниже, в одном полотне).

/login (Вход/Регистрация)
Вкладки «Вход» и «Регистрация». Поля email/пароль. Кнопка «Войти с Google» — disabled (tooltip «Скоро»). Успех → /projects.

/projects
Компактные карточки проектов (название, домен, дата).
Кнопка «Создать проект» — inline-форма сверху (имя, домен) → добавляем в mockApi → сразу открываем /project/:id.

/project/:id — ЕДИНЫЙ ЭКРАН ФЛОУ (без попапов)
Экран — серия аккордеонов/секций (по умолчанию открыты 1–3). Всё компактно (text-sm, плотные таблицы).

1) «Импорт CSV»
Drag&Drop + предпросмотр 20 строк (Papaparse).

Маппинг полей: обязательные url, html_or_text; опциональные title, meta_title, meta_description, pub_date, lang. Мэтчинг — селект над каждой колонкой.

Валидация: дубликаты URL, пустые; счётчики: всего страниц, средняя глубина (эмулируем), сироты (эмулируем).

Кнопка «Обработать» → прямо в секции показываем «Фазы» с прогрессом (эмуляция):
CSV→Parse → HTML-очистка → Разбиение на блоки → «Эмбеддинги» (эмулируем) → Сбор ссылок/глубины → Готово.
Под ним — компактные логи (последние 100 строк) + «Скачать лог» (txt).

2) «Глобальные параметры»
(единые для всех задач; сохраняются в localStorage)

Максимум новых ссылок на страницу (N) — число (по умолчанию 3).

Порядок приоритетов (перетаскиваемые чипы, горизонтально):
[ Хаб ] → [ Коммерция ] → [ Похожие в разделе ] → [ Глубокие ] → [ Свежие ] → [ Сироты ]

Лимиты и правила (компактная сетка):
– Min gap (слова): по умолчанию 200
– Exact-anchor %: по умолчанию 20%
– Stop-anchors (textarea, по одному в строке)
– Old links: Enrich / Regenerate / Audit
– Broken links: Delete / Replace / Ignore
– HTML для ссылок: className (input) + режим append/replace; чекбоксы rel(noopener/noreferrer/nofollow); targetBlank (switch)
– Scope: маска URL (regexp/substring), фильтр «новее даты», случайная выборка %

Важное: никаких настроек каннибализации — убрать полностью.

3) «Быстрые задачи» (выбор задач)
Без кнопок «Запустить» у каждой. Каждая задача — чекбокс/тоггл + минимальные поля.
Запуск — одной общей кнопкой ниже.
Задачи выполняются по очереди в порядке выбранных приоритетов.

Список задач и их минимальные настройки:

Хабы по префиксу (вкл/выкл)
– CSV-загрузка маппинга: prefix, subprefix(optional), hub_url
– Топология: Звезда / Кольцо / Колесо
– Тоггл «Ограничить строго внутри (под)префикса» (вкл)
– Превью-сводка: «префиксов X, страниц Y»

Маршрутизация на коммерческие (money) (вкл/выкл)
– Паттерн URL (regexp, напр. /buy|product|service|pricing/)
– (опц.) Ограничить область по префиксу (input)
– Превью: «найдено ≈ N money-URL»

Похожие в разделе (перекрёстные) (вкл/выкл)
– Префикс(ы)
– k соседей (1=кольцо, 2–3=сетка)

Поднять глубинные (вкл/выкл)
– minDepth (напр. ≥5), доноры уровней ≤3 (чекбокс)

Разогнать свежие (вкл/выкл)
– daysFresh (напр. 30), linksPerDonor (обычно 1)

Подшить сироты (вкл/выкл)
– Область: весь сайт / префикс

Чистка битых ссылок (вкл/выкл)
– Политика 404: Delete / Replace / Ignore

Перегенерация/обогащение (вкл/выкл)
– Режим: Enrich / Regenerate (Regenerate — удаляет старые перед геной)

Примечания по UX:

Выбранные задачи подсвечиваются чипом «Активно».

Под списком задач — Preflight строкой: «Ожидаем примерно: Х ссылок (Хаб ~A, Коммерция ~B, Похожие ~C …)».

4) «Запуск и прогресс»
В этом же экране, ниже задач — одна кнопка «Запустить генерацию».
После клика показываем линейный прогресс и статус-чипы по задачам: Pending / Running / Done / Skipped.
Внутри — компактные строки лога и счётчики: added, rejected, причины (limit_reached, min_gap, exact_exceed, stop_anchor, duplicate, broken_replaced, и т.п.).
По окончании — статус Draft и скроллим к блоку «Черновик».

5) «Черновик (Draft Review)»
Компактная таблица (виртуализация при большом объёме):
Source URL • Target URL • Anchor • Тип (Хаб/Коммерция/Похожие/Глубокие/Свежие/Сироты/404) • Причина (если отклонено) • ✓ Одобрено

Фильтры-чипы сверху по типу. Поиск по URL/анкору.

Inline-diff без модалок: при раскрытии строки показываем кусок очищенного текста с подсветкой <a> (до/после, две узкие колонки side-by-side).

Кнопки внизу: «Одобрить все видимые», «Снять выделение», «Экспорт CSV», «Применить изменения» (эмулируем → статус run = published).

6) «История запусков» (компактный блок)
Плитка прошлых run’ов с метриками: добавлено, 404 исправлено, сироты ↓, глубина ↓, профиль анкоров (Exact/Partial/Generic). Клик → открывает соответствующий черновик/итог в этом же экране (scroll-to).

Поведение генерации (эмуляция логики)
Задачи выполняются последовательно в порядке глобальных приоритетов.

На странице не ставим больше N новых ссылок.

Для каждой задачи сначала пытаемся использовать существующую фразу в подходящем абзаце; если нет — эмулируем «короткий анкор 1–5 слов».

Соблюдаем: Min gap, Exact %, Stop-anchors, Broken policy, Old links (Regenerate чистит перед попытками).

Каннибализации нет (настройки не выводить).

«Похожие в разделе» и «Хабы по префиксу» — строго в границах выбранных (под)префиксов/хаб-CSV.

Компактный стиль (важно)
Tailwind: text-sm, leading-tight, плотные таблицы (divide-y, whitespace-nowrap), карточки без лишних паддингов.

Sticky header + sticky секции «Запуск/Прогресс» и «Черновик» якорями.

Подсказки как title/tooltips, не растягивай экран.

Mock API/Store
Храни: users, projects, settings, pages_*, runs, link_candidates.

SSE/прогресс — через setInterval, меняй проценты и статусы по задачам.

Экспорт CSV — сформируй на клиенте из link_candidates.

Acceptance (Frontend-only)
Авторизация/регистрация (эмуляция), логин-логаут, смена пароля в «Настройках» (эмуляция).

Один экран проекта с 6 секциями: Импорт → Параметры → Задачи → Запуск/Прогресс → Черновик → История.

Выбор задач галочками, одна кнопка «Запустить генерацию», никаких попапов.

Прогресс и логи внутри страницы.

Draft Review: таблица, inline-diff, фильтры, «Одобрить все видимые», экспорт CSV, «Применить».

Нет каннибализации. Хабы — только CSV-маппинг, Money — только паттерн URL.

Если что-то непонятно — спрашивай в комментариях кода. Главное — одна страница, минимализм, чекбоксы задач, одна кнопка запуска.
